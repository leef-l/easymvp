这份清单只做收尾工作：

> 把仓库里仍残留旧词、旧对象名、旧阶段名的低优先级文档一次性列清，并明确为什么它们暂时不再阻塞当前总纲主线。

## 2. 当前结论

当前 EasyMVP 旧 V3 文档已经完成主干收口。

剩余还带旧口径的文件，主要分成两类：

1. 历史推进记录类
2. 低优先级细稿/只读参考类

这些文档仍可能出现：

1. `AcceptanceRun`
2. `production_status / production_passed`
3. `Design / Review / Compile`

但它们现在已不再承担：

1. 当前主入口
2. 当前主契约
3. 当前主页面解释
4. 当前主状态机定义

因此它们属于“可继续保留，但不再阻塞主线”的残留。

## 6. 后续处理原则

后续如果还要继续清文档，建议只遵守这 3 条：

1. 优先清“仍承担主入口/主契约/主解释职责”的文档。
2. 对历史记录类文档，不做大面积语言清洗，只加降级说明即可。
3. 对低优先级细稿，只在它们重新进入实现主链时再修，不为了词面洁癖继续重工。

旧 V3 文档中仍残留的旧词和旧对象名，已经基本被压缩到历史记录和低优先级细稿里。
