## 一、前置依赖检查

### 1.1 必需软件清单

| 软件 | 最低版本 | 验证命令 | 用途 |
|------|---------|---------|------|
| **Node.js** | v20+ (推荐 v24 LTS) | `node -v` | Electron + Vite 构建 |
| **pnpm** | v8+ (推荐 v9+) | `pnpm -v` | 包管理（项目使用 `pnpm-lock.yaml`） |
| **Go** | 1.24.0 | `go version` | 编译 `easymvp-core` 二进制 |
| **Git** | 任意 | `git --version` | 拉取代码 |

### 1.2 Windows 特定工具链

| 场景 | 需要安装 | 说明 |
|------|---------|------|
| `pnpm package:dir`（目录模式） | ❌ 无需额外工具 | 仅生成 `win-unpacked` 文件夹 |
| `pnpm package`（完整安装包） | **NSIS**（可选） | 生成 `.exe` 安装程序 |
| `pnpm package`（完整安装包） | **WiX Toolset**（可选） | 生成 `.msi` 安装程序（未在配置中启用） |

> **结论**：如果只是验证打包（`package:dir`），不需要安装 NSIS 或 WiX。electron-builder 会自动处理 `dir` 模式。

### 1.3 快速安装缺失依赖

```powershell
# 安装 Node.js (via fnm 或 nvm-windows，或直接下载安装包)
# 推荐: https://nodejs.org/ 下载 LTS 版本

# 安装 pnpm
npm install -g pnpm

# 安装 Go (如果缺失)
# 推荐: https://go.dev/dl/ 下载 go1.24.0.windows-amd64.msi
# 或 winget
winget install GoLang.Go
```

## 二、`package.json` 打包 Scripts 说明

文件位置：`apps/desktop/package.json`

| Script | 命令 | 作用 |
|--------|------|------|
| `build:core` | `node ./scripts/build-core.mjs` | 编译 Go core → `resources/bin/easymvp-core.exe` |
| `build` | `tsc -p tsconfig.node.json && vite build && tsc -p tsconfig.json` | 编译 TS + 构建前端 + 类型检查 |
| `package` | `pnpm run build:core && pnpm run build && electron-builder --publish never` | **完整打包**（生成安装程序） |
| `package:dir` | `pnpm run build:core && pnpm run build && electron-builder --dir --publish never` | **仅生成目录**（推荐用于验证） |
| `verify:package` | `node ./scripts/verify-package.mjs` | 验证打包产物 + Smoke 测试 |

### 2.1 electron-builder 配置摘要

```json
{
  "appId": "com.easymvp.desktop",
  "productName": "EasyMVP",
  "directories": { "output": "release" },
  "win": {
    "target": [
      { "target": "nsis", "arch": ["x64"] },
      { "target": "portable", "arch": ["x64"] }
    ]
  }
}
```

- **输出目录**：`apps/desktop/release/`
- **Windows 产物**：
  - `release/win-unpacked/` — 解压后的应用目录（`package:dir` 生成）
  - `release/EasyMVP Setup x.x.x.exe` — NSIS 安装程序（`package` 生成，需 NSIS）
  - `release/EasyMVP x.x.x.exe` — Portable 单文件（`package` 生成）

## 四、`verify-package.mjs` 断言内容详解

文件位置：`apps/desktop/scripts/verify-package.mjs`

### 4.1 文件系统断言（打包前检查）

| 检查项 | 路径 | 含义 |
|--------|------|------|
| releaseRoot 存在 | `release/win-unpacked/` | 打包输出目录已生成 |
| binaryPath 存在 | `release/win-unpacked/resources/bin/easymvp-core.exe` | Go 核心二进制已正确嵌入 |
| migrationPath 存在 | `release/win-unpacked/resources/manifest/migrations/` | 数据库迁移文件已嵌入 |
| executablePath 存在 | `release/win-unpacked/EasyMVP.exe` | 主可执行文件已生成 |

### 4.2 Smoke 测试断言（运行时检查）

验证脚本启动 Electron 应用并捕获输出日志：

| 断言 | 检查内容 | 失败含义 |
|------|---------|---------|
| `managed=true` | 核心进程受 Desktop 管理 | Core 启动失败或未接入 IPC |
| `phase=ready` | 应用生命周期到达 ready 阶段 | 初始化流程异常中断 |
| `baseUrl` 匹配 | 核心服务实际监听地址与预期一致 | 端口绑定异常 |
| `corePid > 0` | 核心进程 PID 有效 | Core 进程未启动或崩溃 |
| `healthOk=true` + HTTP 200 + `/api/v3/system/healthz` | 核心 HTTP 健康接口可用 | Go Core HTTP 服务异常 |
| `rendererMounted=true` + `rootChildCount > 0` | React 渲染器已挂载 DOM | 前端渲染失败（白屏） |
| `cleanup=stopped` | 应用退出时核心正确清理 | 进程残留 / 资源泄漏 |

### 4.3 环境变量（验证时自动设置）

```
EASYMVP_MANAGE_CORE=1           # Desktop 管理 Core 生命周期
EASYMVP_CORE_BASE_URL=...       # 动态分配的本地端口
EASYMVP_CORE_DATA_ROOT=...      # 临时数据目录
ELECTRON_DISABLE_SANDBOX=1      # 禁用 sandbox（验证环境必需）
EASYMVP_DESKTOP_SMOKE=1         # Smoke 测试模式标识
```

> **超时**：Smoke 测试有 **45 秒** 超时限制。

## 六、常见坑与解决方案

### 坑 1：`pnpm run package:dir` 失败 — `go build` 找不到 main.go

**现象**：`build:core` 报错 `go build failed`

**原因**：`build-core.mjs` 通过 `cwd: coreRoot` 在 `apps/core` 目录执行 `go build -o ... ./main.go`。需要确保 `apps/core/main.go` 存在。

**解决**：
```powershell
# 检查 main.go 是否存在
test-path C:\Users\Public\project\easymvp\apps\core\main.go
# 如果不存在，从 git 恢复或检查文件路径
```

### 坑 2：`pnpm run package` 失败 — `cannot find module 'electron-builder-squirrel-windows'`

**现象**：electron-builder 报错找不到某个 target 的依赖

**原因**：项目配置中只使用了 `nsis` 和 `portable`，但如果全局配置有残留引用会报错。

**解决**：本项目配置正确，如仍报错请检查 `package.json` 的 `build.win.target` 字段，确保没有配置未安装的工具。

### 坑 3：`verify:package` 失败 — `no packaged desktop directory build found`

**现象**：找不到 `release/win-unpacked`

**原因**：你运行了 `pnpm run package` 而非 `pnpm run package:dir`，或者打包失败中断了。

**解决**：
```powershell
# 确认目录存在
test-path C:\Users\Public\project\easymvp\apps\desktop\release\win-unpacked

# 如果不存在，重新运行 package:dir
cd apps\desktop
pnpm run package:dir
```

### 坑 4：`verify:package` Smoke 测试超时（45秒）

**现象**：`desktop smoke test timed out after 45000ms`

**可能原因**：
1. **首次运行**：Electron 应用首次启动较慢
2. **端口占用**：`allocateSmokePort` 分配的端口被占用
3. **杀毒软件拦截**：Windows Defender 或其他杀毒软件拦截了 `easymvp-core.exe` 启动
4. **Go Core 编译失败**：`easymvp-core.exe` 不完整或损坏

**解决**：
```powershell
# 1. 手动测试 core 是否能独立启动
cd C:\Users\Public\project\easymvp\apps\desktop\release\win-unpacked\resources\bin
.\easymvp-core.exe --help

# 2. 检查 Windows Defender 是否隔离了 easymvp-core.exe
# 打开 Windows 安全中心 → 病毒和威胁防护 → 保护历史记录

# 3. 手动运行 EasyMVP.exe 观察是否有弹窗报错
cd C:\Users\Public\project\easymvp\apps\desktop\release\win-unpacked
.\EasyMVP.exe
```

### 坑 5：`pnpm install --frozen-lockfile` 失败

**现象**：lockfile 与 package.json 不匹配

**解决**：
```powershell
# 方案 A：更新 lockfile
pnpm install

# 方案 B：如果确定 package.json 未改动，强制使用当前 lockfile
pnpm install --frozen-lockfile --prefer-offline
```

### 坑 6：Node 版本过低导致 TypeScript/Vite 构建失败

**现象**：`vite build` 或 `tsc` 报错语法不支持

**解决**：
```powershell
# 检查版本
node -v  # 需要 v20+

# 升级
# 推荐通过 nvm-windows 或 fnm 管理
nvm install 22
nvm use 22
```

### 坑 7：`electron-builder` 下载依赖慢 / 卡住

**现象**：打包过程中长时间卡在 `downloading` 某步骤

**原因**：electron-builder 需要下载 Electron 二进制、签名工具等，默认走 GitHub Releases，国内网络慢。

**解决**：设置镜像源
```powershell
# PowerShell 临时设置（当前会话有效）
$env:ELECTRON_BUILDER_CACHE="$env:LOCALAPPDATA\electron-builder\Cache"
$env:ELECTRON_MIRROR="https://npmmirror.com/mirrors/electron/"

# 或者在用户级永久设置（推荐）
[Environment]::SetEnvironmentVariable("ELECTRON_MIRROR", "https://npmmirror.com/mirrors/electron/", "User")
```
