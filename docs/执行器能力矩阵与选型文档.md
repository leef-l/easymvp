# 执行器能力矩阵与选型文档

## 一、目标

给平台接入执行器提供统一选型依据。

---

## 二、候选执行器

1. Aider
2. OpenHands
3. Claude Code
4. Codex CLI
5. Gemini CLI
6. Chat

---

## 三、能力矩阵

| 执行器 | 类型 | 适合任务 | 多文件编辑 | 自主探索 | 接入难度 | 当前优先级 |
|---|---|---|---|---|---|---|
| Aider | CLI | 编码实现 | 强 | 中 | 低 | 已接入 |
| OpenHands | Agent | 复杂多步任务 | 强 | 强 | 中 | 高 |
| Claude Code | CLI | 重构/编码 | 强 | 中 | 中 | 高 |
| Codex CLI | CLI | 编码/脚本 | 强 | 中 | 中 | 高 |
| Gemini CLI | CLI | 编码/理解 | 中 | 中 | 中 | 中 |
| Chat | API | 轻任务/分析/创作 | 弱 | 弱 | 低 | 已接入 |

---

## 四、推荐场景

### 4.1 Aider

适合：

- 文件明确
- 目标明确
- 快速代码修改

### 4.2 OpenHands

适合：

- 需要探索项目
- 需要多步操作
- 需要沙箱运行和验证

### 4.3 Claude Code / Codex CLI / Gemini CLI

适合：

- 仓库级改动
- 高质量编码任务
- 跨文件重构

### 4.4 Chat

适合：

- 设计
- 分析
- 审核
- 文案/创作

---

## 五、接入建议

第一优先级：

1. OpenHands
2. Claude Code
3. Codex CLI

第二优先级：

4. Gemini CLI

---

## 六、结论

EasyMVP 应优先接入“可无头执行、可批量调度、可稳定回写”的执行器，而不是优先接入 IDE 交互工具。
