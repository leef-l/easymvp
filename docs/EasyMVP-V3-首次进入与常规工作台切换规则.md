# EasyMVP V3 首次进入与常规工作台切换规则

> 更新时间：2026-04-19
> 上游文档：[EasyMVP-V3-创建后首次进入Project-Workspace引导态设计](./EasyMVP-V3-创建后首次进入Project-Workspace引导态设计.md)
> 关联文档：[EasyMVP-V3-首次进入工作台线框图设计](./EasyMVP-V3-首次进入工作台线框图设计.md)
> 目标：定义首次进入引导态何时进入、何时退出，以及退出后如何切换到常规工作台。

## 1. 进入条件

建议同时满足：

1. `is_first_entry_guided = true`
2. 项目处于早期 `reviewing`
3. 尚未产生稳定执行活动

## 2. 退出条件

任一满足即可：

1. 用户手动关闭
2. 用户进入过 `Plan`
3. 项目进入稳定 `reviewing` 或开始进入 `executing`
4. 出现正式执行活动

## 3. 切换方式

建议：

1. 不整页刷新
2. 平滑切换模块优先级
3. 保留事件流上下文

## 4. 不该怎么做

不应该：

1. 每次进项目都重新进入引导态
2. 切换时丢失当前上下文
