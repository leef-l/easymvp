# 执行器结果回写与日志事件协议文档

## 一、目标

统一执行器运行结果、流输出、错误、产物的回写规则。

---

## 二、回写对象

1. `mvp_domain_task`
2. `mvp_task_log`
3. `mvp_workflow_event`
4. 后续建议新增 `mvp_execution_session`

---

## 三、统一回写字段

### DomainTask

- `status`
- `result`
- `started_at`
- `completed_at`
- `updated_at`
- `heartbeat_at`

### TaskLog

- 启动
- 失败
- 重试
- 取消
- 手动恢复

### WorkflowEvent

- 执行器启动
- 执行器输出
- 执行器结束

---

## 四、事件协议

建议事件：

- `executor.started`
- `executor.output`
- `executor.error`
- `executor.completed`
- `executor.canceled`

---

## 五、输出规范

1. `stdout/stderr` 分开存
2. 日志支持截断
3. 长输出保留摘要 + 原始附件
