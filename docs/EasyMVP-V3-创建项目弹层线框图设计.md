# EasyMVP V3 创建项目弹层线框图设计

> 更新时间：2026-04-19
> 上游文档：[EasyMVP-V3-创建项目流程与页面设计](./EasyMVP-V3-创建项目流程与页面设计.md)
> 关联文档：[EasyMVP-V3-创建初始化事件流接口设计](./EasyMVP-V3-创建初始化事件流接口设计.md)
> 关联文档：[EasyMVP-V3-工作台全页面设计规范](./EasyMVP-V3-工作台全页面设计规范.md)
> 目标：把 `New Project` 弹层推进到线框图级别，明确桌面端结构、状态切换、初始化态布局和失败态表现。

## 1. 设计结论

`New Project` 不能是传统表单弹窗。

它必须像一个“轻量启动面板”：

1. 看起来轻
2. 填起来快
3. 系统自动推导看得见
4. 创建过程实时可见

## 2. 默认尺寸与容器

建议桌面端默认使用大尺寸居中弹层。

推荐尺寸：

1. 宽度 `920-1040px`
2. 高度不超过视口 `82vh`
3. 内部滚动，不滚页面背景

原因：

1. 需要放下 4 段信息
2. 仍然保持“附着在 Workspace 上”的感觉
3. 不要切换成全新页面打断上下文

## 4. 布局规则

### 4.1 顶部

顶部只放：

1. 标题
2. 一句副标题
3. 关闭按钮

不放：

1. 二级标签页
2. 复杂帮助导航
3. 额外统计

### 4.2 主体区

主体区固定为三段：

1. 项目与目标
2. 工作区路径
3. 自动推导预览

### 4.3 底部

底部只保留两个动作：

1. `Cancel`
2. `Create Project`

主按钮必须始终在右下角固定可见。

## 5. 分类卡线框

分类不应是下拉框。

建议使用横向或两行卡片：

```text
┌────────────┐  ┌────────────┐  ┌────────────┐
│ Web App    │  │ Game       │  │ Video Edit │
│ UI + API   │  │ Runtime    │  │ Editor     │
└────────────┘  └────────────┘  └────────────┘
```

每张卡只放：

1. 分类名
2. 一句场景说明

选中态必须非常明确。

## 6. 路径选择区线框

```text
┌────────────────────────────────────────────────────────────────────┐
│ Workspace Source                                                  │
│ [Create New Workspace] [Use Existing Folder] [Use Repository]     │
│ [ /path/to/project............................................ ]  │
│ [Browse]                                                          │
│ ✓ Writable   ✓ Git Repo   ! Large node_modules detected           │
└────────────────────────────────────────────────────────────────────┘
```

路径检查结果必须贴近输入框显示，而不是藏在次级详情里。

## 7. 自动推导预览区线框

```text
┌────────────────────────────────────────────────────────────────────┐
│ EasyMVP Will Prepare                                              │
│ ┌──────────────┐ ┌──────────────┐ ┌──────────────┐ ┌─────────────┐ │
│ │ Planning     │ │ Execution    │ │ Acceptance   │ │ Workspace   │ │
│ │ Draft->...   │ │ Auto resolve │ │ Category     │ │ Local root  │ │
│ └──────────────┘ └──────────────┘ └──────────────┘ └─────────────┘ │
└────────────────────────────────────────────────────────────────────┘
```

这个区是解释区，不是配置区。

## 9. 失败态线框

```text
┌────────────────────────────────────────────────────────────────────┐
│ Create Failed                                               [Close]│
├────────────────────────────────────────────────────────────────────┤
│ Failed at: binding_profiles                                        │
│ Reason: default category profile binding failed                    │
│ Suggested action: check profile config and retry                   │
├────────────────────────────────────────────────────────────────────┤
│ Completed Steps                                                    │
│ ✓ project record created                                           │
│ ✓ workspace initialized                                            │
│ ✕ profiles not bound                                               │
├────────────────────────────────────────────────────────────────────┤
│ [Edit Inputs]                                    [Retry Creation] │
└────────────────────────────────────────────────────────────────────┘
```

失败态不应该把用户丢回空白草稿态。

## 10. 响应式降级

### 10.1 平板

建议：

1. 由四列解释卡降为两列
2. 路径区与推导区纵向堆叠

### 10.2 窄屏

建议转为全屏抽屉或全屏页。

原因：

1. 创建项目不是极短微交互
2. 小屏硬塞弹层会破坏可读性

## 11. 视觉规则

建议：

1. 白底或极浅灰底
2. 标题与主按钮形成明确锚点
3. 卡片轻边框、轻阴影
4. 避免多色块背景

这不是营销弹层，而是工作流启动器。

## 12. 不该怎么做

创建项目弹层不应该：

1. 做成窄小 modal
2. 放十几个表单字段
3. 把初始化过程隐藏成 spinner
4. 失败后只显示技术错误
5. 让主按钮滚出视口
