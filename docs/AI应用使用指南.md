# EasyMVP AI 应用使用指南

> 更新日期：2026-04-08  
> **⚠️ 状态：待废弃** — 本文描述的是旧版 AI 管理模块（供应商/套餐/模型/引擎/手工任务）。MACCS v2 闭环工作流落地后，角色配置与任务执行将由中央大脑统一编排，本指南将被 [`MACCS-EasyMVP-闭环工作流规范.md`](MACCS-EasyMVP-闭环工作流规范.md) 取代。

本文只覆盖当前代码中仍在使用的 AI 管理能力，并说明它与 MVP 项目的关系。

## 模块关系

当前 AI 模块分为 5 类配置：

1. `供应商`：AI 服务来源与 Base URL
2. `套餐`：API Key / Secret
3. `模型`：实际调用的模型与项目角色能力
4. `引擎`：Aider / OpenHands 等执行引擎参数
5. `任务`：手工发起的一次性 AI 执行任务

MVP 项目不会绕过这些配置。`MVP -> 系统概览` 会直接检查其中的关键项是否可用。

## 推荐配置顺序

### 1. 配置 AI 供应商

入口：`AI 管理 -> 供应商管理`

关键字段：

- `供应商名称`
- `供应商代码`
- `Provider 类型`
- `API 基础地址`
- `状态`

当前前端会根据 `baseURL` 自动识别常见协议：

- 包含 `anthropic` 路径时走 Anthropic 风格
- 其他大多数模型供应商走 OpenAI 兼容风格

### 2. 配置套餐

入口：`AI 管理 -> 套餐管理`

套餐用于保存密钥，并关联到供应商。

关键字段：

- `供应商ID`
- `套餐名称`
- `套餐代码`
- `API Key`
- `API Secret`（部分供应商需要）

### 3. 配置模型

入口：`AI 管理 -> 模型管理`

当前模型配置里最重要的不是旧文档中的 `chat / reasoning / coding`，而是项目角色能力：

- `architect`
- `implementer`
- `auditor`
- `coordinator`

常用字段：

- `套餐ID`
- `供应商ID`
- `模型显示名称`
- `模型代码`
- `项目角色`
- `最大输出 token`
- `上下文窗口大小`
- `是否支持流式输出`
- `默认角色提示词`

如果你准备让模型参与 MVP 项目，至少要保证：

- 架构师有可用模型
- 实现者有可用模型
- 审核者有可用模型

### 4. 配置执行引擎

入口：`AI 管理 -> 执行引擎`

当前代码里有独立的引擎配置页，用于维护 Aider / OpenHands 这类执行器。

可配置项包括：

- `默认模型`
- `超时秒数`
- `最大步数`
- `工作区根目录`
- `命令模板`
- `额外 JSON 配置`
- `启用状态`

当前页面还支持“测试连接”，建议在把引擎交给 MVP 之前先测通。

说明：

- 这里的引擎参数主要服务于 `AI 管理 -> 任务` 的手工执行能力
- MVP 项目仍然会结合 `MVP -> 配置管理`、项目 `workDir` 和角色预设共同决定运行行为

### 5. 配置 MVP 角色预设

入口：`MVP -> 角色预设`

角色预设把“角色类型 + 角色等级 + 模型 + 执行方式”绑定在一起，是 MVP 项目的默认模板。

常见组合：

- `architect / max`
- `implementer / lite|pro|max`
- `auditor / pro`
- `coordinator / lite`

如果角色预设没准备好，项目即使能创建，也很容易在审核或执行阶段缺模型。

## 手工任务执行

入口：`AI 管理 -> 任务`

这个页面适合做一次性验证或脱离 MVP 的手工执行任务。当前表单支持：

- `执行引擎`：`aider` / `openhands`
- `仓库路径`
- `工作目录`
- `分支名`
- `任务指令`

适用场景：

- 验证某个引擎是否可写仓
- 验证模型与执行器组合是否正常
- 单独做一次小规模代码变更

## 与 MVP 的关系

当你在 MVP 中创建项目时，真正生效的是下面这一整套组合：

1. AI 供应商、套餐、模型是否可用
2. 角色预设是否完整
3. 执行引擎是否可用
4. 项目的 `workDir` 是否有效
5. `MVP -> 配置管理` 中的全局参数是否合理

建议先到 `MVP -> 系统概览` 查看检查结果，再正式发起项目。

## 常见问题

### 1. 模型已经创建，为什么项目还是跑不起来？

常见原因不是“没有模型”，而是“模型没有分配到正确的项目角色”，或者角色预设没有配置完整。

### 2. `Aider` / `OpenHands` 一直失败怎么办？

先在 `AI 管理 -> 执行引擎` 里检查默认模型、工作区、命令模板和连接测试，再去看 `AI 管理 -> 任务` 的执行日志。

### 3. 支持本地模型吗？

支持。只要你的服务提供 OpenAI 兼容接口，就可以按供应商 / 套餐 / 模型的方式接入。

### 4. 配好 AI 模块后，还要再配 MVP 吗？

要。AI 模块解决的是“模型和执行器能不能被调用”，MVP 模块还需要项目分类、角色预设、工作目录和工作流配置。
