# EasyMVP V3 Plan 线框图设计

> 更新时间：2026-04-19
> 上游文档：[EasyMVP-V3-Plan详细页面设计](./EasyMVP-V3-Plan详细页面设计.md)
> 关联文档：[EasyMVP-V3-Plan-diff组件规范](./EasyMVP-V3-Plan-diff组件规范.md)
> 目标：把 `Plan` 页推进到线框图级别，明确三栏对照、任务投影区和差异区的视觉结构。

## 1. 设计结论

`Plan` 页必须看起来像“决策解释台”，不是普通任务页。

## 2. 线框结构

```text
┌────────────────────────────────────────────────────────────────────┐
│ Plan Overview                                                     │
├─────────────────────┬─────────────────────┬────────────────────────┤
│ Draft               │ Review              │ Compiled               │
│ goal / tasks        │ blockers / advice   │ final tasks/contracts  │
├────────────────────────────────────────────────────────────────────┤
│ Task Projection                                                     │
├────────────────────────────────────────────────────────────────────┤
│ Diff & Reasoning                                                    │
└────────────────────────────────────────────────────────────────────┘
```

## 3. 核心规则

1. 三栏必须同屏可比
2. 差异区不能被折叠成次要信息
3. 任务投影区必须解释来源关系

## 4. 三栏比例建议

建议：

1. `Draft 30%`
2. `Review 32%`
3. `Compiled 38%`

## 5. 不该怎么做

不应该：

1. 只有一个任务表
2. 差异区只放在抽屉里
3. review 和 compiled 不能对照

## 6. 后续细分专题

1. Plan 任务投影抽屉线框
2. 三栏联动滚动规则
