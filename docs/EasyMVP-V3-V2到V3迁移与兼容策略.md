# EasyMVP V3 V2 到 V3 迁移与兼容策略

> 更新时间：2026-04-19
> 上游文档：[EasyMVP-V3总体架构设计](./EasyMVP-V3总体架构设计.md)
> 关联文档：[EasyMVP-V3计划数据模型与表结构设计](./EasyMVP-V3计划数据模型与表结构设计.md)
> 目标：定义 V2 过渡到 V3 的迁移阶段、兼容策略和风险控制方式。

## 1. 设计结论

V3 不应一次性硬切。

建议分三阶段迁移：

1. 执行底座替换
2. 计划链路前置
3. 验收与工作台升级

## 2. Phase 1

目标：

1. 先接 `brain-v3`
2. 保持原有主链路基本不动
3. 拿到 run 生命周期

## 3. Phase 2

目标：

1. 引入 `PlanDraft / PlanReviewResult / CompiledPlan`
2. 阻止 architect 方案直接落任务
3. 引入 `easymvp-brain`

## 4. Phase 3

目标：

1. 验收升级为生产级
2. 工作台升级为实时驾驶舱
3. 最终状态从 `executor success` 升级为 `production_passed`

## 5. 兼容原则

建议：

1. 老项目可继续按 V2 路径运行
2. 新项目默认走 V3
3. 中间态项目不强制迁移历史数据

## 6. 风险

主要风险：

1. 状态机双轨复杂度
2. 计划对象与旧任务对象并存
3. 验收口径切换

## 7. 后续细分专题

本专题后续继续拆：

1. 双轨状态机设计
2. 老数据兼容字段策略
3. 迁移开关设计
