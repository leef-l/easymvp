# 飞书机器人完整接入设计文档

> 目标：通过飞书机器人直接管理 EasyMVP 项目，支持创建项目、查看状态、审批操作，双模式接入（Webhook 回调 + WebSocket 长连接）。

---

## 一、整体架构

```
飞书用户
  │  发消息 / 点卡片按钮
  ▼
飞书平台
  │  Webhook POST 或 WebSocket 推送
  ▼
EasyMVP 后端
  ├── feishu_callback.go     Webhook 回调入口（已有，补注册）
  ├── feishu_ws.go           WebSocket 长连接管理（新增）
  ├── feishu_bot.go          消息解析 + 指令路由（新增）
  └── feishu_reply.go        回复消息（文本/卡片）（新增）
  │
  ▼
业务层（已有）
  ├── workflow.go            CreateProject / ConfirmPlan / Pause / Resume
  ├── chat_engine.go         SendMessage
  └── scheduler.go           任务调度
```

## 三、双模式接入

### 3.1 Webhook 模式（需公网地址）

```
飞书 → POST https://your-domain/api/mvp/collab/feishu/callback
     → 验证签名（X-Lark-Request-Timestamp + X-Lark-Request-Nonce + X-Lark-Signature）
     → 解析事件类型
     → 路由到对应处理器
```

**配置项**：App ID、App Secret、Verification Token、Encrypt Key、回调地址

**已有问题**：回调处理逻辑已写，但路由未注册到 cmd_custom.go（P0 修复）

### 3.2 WebSocket 长连接模式（无需公网）

```
EasyMVP 后端 → 主动连接飞书 WSS 端点
              → 获取 WebSocket URL（飞书 API）
              → 保持长连接 + 心跳（30s ping）
              → 断线自动重连（指数退避，最大 5 分钟）
              → 接收事件推送，路由到相同处理器
```

**配置项**：App ID、App Secret（无需 Verification Token / Encrypt Key / 公网地址）

**实现方案**：
- 使用 Go 标准库 `golang.org/x/net/websocket` 或 `github.com/gorilla/websocket`
- 独立 goroutine 维护连接，应用启动时按配置自动启动
- WebSocket URL 通过飞书 API 获取：`POST /open-apis/event/v1/app_ticket/resend`（获取 WS endpoint）
  - 实际端点：`wss://open.feishu.cn/event/v2/websocket/connect?app_id=xxx`（飞书 WSS SDK 协议）

## 五、事件处理流程

### 5.1 消息事件（im.message.receive_v1）

```
收到事件
  │
  ├─ 解析 event.message.content（JSON 字符串）
  ├─ 去除 @ 提及符（如 @EasyMVP）
  ├─ 提取指令文本
  │
  └─ 路由
       ├─ "创建项目" → handleCreateProject()
       ├─ "项目列表" → handleListProjects()
       ├─ "项目状态" → handleProjectStatus()
       ├─ "暂停"     → handlePauseProject()
       ├─ "继续"     → handleResumeProject()
       └─ 其他       → handleHelp()
```

### 5.2 卡片交互事件（card.action.trigger）

```
收到事件
  │
  ├─ 解析 action.value（包含 action_id、action、project_id 等）
  ├─ 反查 open_id → system_user_id
  └─ 调用 DecisionCenter.ApproveAction() / RejectAction()
```

## 六、指令处理细节

### 创建项目

**输入**：`创建项目 电商后台` 或 `创建项目 H5游戏 分类:游戏开发`

**处理**：
1. 解析项目名称和分类（默认分类：软件开发）
2. 从绑定表反查 system_user_id（未绑定则回复引导绑定）
3. 调用 `workflow.CreateProject()`（使用该用户身份）
4. 回复项目创建成功卡片（含项目ID、工作目录、下一步提示）

**回复卡片**：
```
✅ 项目已创建
───────────────
📁 项目名称：电商后台
🏷️ 分类：软件开发
🆔 项目ID：315xxx
───────────────
下一步：在 EasyMVP 管理端与架构师对话，
确认方案后即可自动执行。
[打开管理端] [查看项目状态]
```

### 项目列表

**回复**：文本消息，列出最近 5 个项目（名称、状态、进度百分比）

### 项目状态

**回复**：卡片，显示任务总数、已完成、运行中、失败数

## 七、WebSocket 实现细节

### 7.1 获取 WS 连接地址

飞书长连接协议需要先调 HTTP API 获取 endpoint：

```
POST https://open.feishu.cn/open-apis/event/v1/app_ticket/resend
Header: Authorization: Bearer {tenant_access_token}
```

实际 WS 端点（飞书官方 Go SDK 协议）：
```
wss://open.feishu.cn/event/v2/websocket/connect?app_id={app_id}&token={wss_token}
```

### 7.2 连接管理（feishu_ws.go）

```go
type FeishuWSClient struct {
    appID     string
    appSecret string
    conn      *websocket.Conn
    handler   EventHandler  // 与 Webhook 模式共用同一 handler
    stopCh    chan struct{}
    mu        sync.Mutex
}

func (c *FeishuWSClient) Start()    // 启动连接 + 重连循环
func (c *FeishuWSClient) Stop()     // 关闭连接
func (c *FeishuWSClient) connect()  // 建立 WS 连接
func (c *FeishuWSClient) readLoop() // 读取事件推送
func (c *FeishuWSClient) pingLoop() // 30s 心跳
```

### 7.3 重连策略

- 初始重连间隔：5s
- 指数退避：5s → 10s → 20s → 40s → 最大 300s
- 重连成功后重置间隔
