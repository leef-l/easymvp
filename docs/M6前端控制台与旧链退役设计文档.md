# M6：前端控制台与旧链退役设计文档

> 已归档：本文记录的是 Workflow V2 切换期的实施方案。当前代码已默认使用 Workflow V2，文中关于 `legacy` 分流、待做事项和里程碑拆解不再代表现状。
>
> 文档定位：M6 里程碑实施设计
>
> 目标：新项目默认走 workflow_v2，旧链降为兼容只读模式，补齐 Execution Console。

---

## 一、M6 目标

1. 新项目创建时**默认使用 workflow_v2**，不再需要手动选择引擎版本
2. 现有 legacy 项目保持可用（只读兼容），但不再允许新建
3. 补齐 **Execution Console**（执行控制台），提供领域任务实时监控
4. 项目列表统一展示新旧项目，legacy 项目标记为旧引擎

## 三、实施方案

### 3.1 阶段 A：新项目强制 V2（后端）

**改动点**：

1. **`engine/workflow.go:179-181`**：默认值 `"legacy"` → `"workflow_v2"`
   ```go
   ev := "workflow_v2"  // 不再默认 legacy
   if len(engineVersion) > 0 && engineVersion[0] == "legacy" {
       ev = "legacy"  // 仅允许显式指定 legacy（兼容旧调用）
   }
   ```

2. **`controller/chat/workflow.go:54`**：CreateProject 始终执行 V2 初始化
   - 移除 `if req.EngineVersion == "workflow_v2"` 判断
   - 始终调用 `CreateRun` + `StartDesign`
   - 如果前端显式传 `legacy`，跳过 V2 初始化

3. **`v1/workflow.go`**：`EngineVersion` 字段保留但标注废弃
   - 不传时默认 `workflow_v2`
   - 传 `legacy` 仍然可用（兼容期）

### 3.2 阶段 B：简化 controller 分流（后端）

**原则**：V2 为主路径（不缩进），legacy 为兼容分支（缩进包在 if 块内）。

当前所有 handler 的结构是：
```go
if isV2 {
    // V2 逻辑（主路径）
} else {
    // legacy 逻辑
}
```

改为：
```go
if !isV2 {
    // legacy 兼容（未来删除）
    return legacyResult, nil
}
// V2 主路径（不缩进）
```

这样未来删除 legacy 时只需删掉 if 块。

### 3.3 阶段 C：Execution Console API（后端）

新增 3 个 API 端点：

| 方法 | 路径 | 用途 |
|------|------|------|
| GET | `/mvp/workflow/execution-status` | 执行阶段实时状态（任务列表、批次进度、资源锁） |
| GET | `/mvp/workflow/domain-tasks` | 领域任务列表（支持按 workflowRunID/status 筛选） |
| GET | `/mvp/workflow/resource-locks` | 当前资源锁列表（调试用） |

#### execution-status 响应结构

```go
type WorkflowExecutionStatusRes struct {
    WorkflowRunID  snowflake.JsonInt64   `json:"workflowRunID"`
    StageRunID     snowflake.JsonInt64   `json:"stageRunID"`
    StageStatus    string                `json:"stageStatus"`
    ActiveBatch    int                   `json:"activeBatch"`
    TotalTasks     int                   `json:"totalTasks"`
    CompletedTasks int                   `json:"completedTasks"`
    RunningTasks   int                   `json:"runningTasks"`
    FailedTasks    int                   `json:"failedTasks"`
    PendingTasks   int                   `json:"pendingTasks"`
    Tasks          []DomainTaskItem      `json:"tasks"`
    ResourceLocks  []ResourceLockItem    `json:"resourceLocks"`
}

type DomainTaskItem struct {
    ID               snowflake.JsonInt64 `json:"id"`
    Name             string              `json:"name"`
    Status           string              `json:"status"`
    RoleType         string              `json:"roleType"`
    BatchNo          int                 `json:"batchNo"`
    ExecutionMode    string              `json:"executionMode"`
    AffectedResources []string           `json:"affectedResources"`
    StartedAt        *gtime.Time         `json:"startedAt,omitempty"`
    CompletedAt      *gtime.Time         `json:"completedAt,omitempty"`
    ErrorMessage     string              `json:"errorMessage,omitempty"`
    RetryCount       int                 `json:"retryCount"`
}

type ResourceLockItem struct {
    Resource string              `json:"resource"`
    TaskID   snowflake.JsonInt64 `json:"taskID"`
    TaskName string              `json:"taskName"`
}
```

#### domain-tasks 响应结构

```go
type WorkflowDomainTasksReq struct {
    ProjectID snowflake.JsonInt64 `json:"projectID" v:"required"`
    Status    string              `json:"status"`    // 可选筛选
    BatchNo   int                 `json:"batchNo"`   // 可选筛选
}

type WorkflowDomainTasksRes struct {
    Tasks []DomainTaskItem `json:"tasks"`
    Total int              `json:"total"`
}
```

### 3.4 阶段 D：前端改动

1. **项目创建表单**：`engineVersion` 默认值改为 `workflow_v2`，选择器保留但标注旧引擎为"(不推荐)"
2. **项目列表**：legacy 项目显示灰色 Tag，V2 项目不再显示引擎版本（减少噪音）
3. **Execution Console 页面**：新增 `/mvp/workflow/execution` 页面
   - 批次进度条（Batch Board）
   - 任务卡片列表（按批次分组）
   - 资源锁面板
   - 任务操作按钮（重试/跳过）

## 四、不做的事

1. **不删除 legacy 代码**：M6 只是降级，不是删除。legacy 分支保留以兼容现有项目。
2. **不迁移旧项目数据**：现有 legacy 项目继续使用旧表，不做 mvp_task → mvp_domain_task 迁移。
3. **不删除 engine/scheduler.go 等旧文件**：旧项目仍可能在运行中。
4. **不做执行器统一抽象**：那是半年路线图月 3 的内容。
