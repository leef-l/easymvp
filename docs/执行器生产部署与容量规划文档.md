# 执行器生产部署与容量规划文档

## 一、目标

指导执行器上线部署与容量规划。

---

## 二、部署模式

### 2.1 单机混跑

适合早期验证：

- Chat
- Aider
- 少量 OpenHands

### 2.2 分层部署

建议中期：

- Web/API
- Scheduler/Worker
- OpenHands Runtime

### 2.3 执行器专属节点

建议后期：

- Aider Worker
- OpenHands Worker
- Chat Worker

---

## 三、容量建议

9950X + 48G 环境初始建议：

- Chat：20-40
- Aider：6-12
- OpenHands：2-4
- 混跑重执行器：4-8

---

## 四、监控指标

1. 执行器启动成功率
2. 平均耗时
3. 并发占用
4. 失败分类分布
5. 超时率

---

## 五、扩容信号

满足任一项建议扩容：

1. 任务排队时间持续升高
2. OpenHands 长尾任务积压
3. Aider 频繁超时
4. CPU 长期高占用
5. IO 等待持续高
