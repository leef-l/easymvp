# EasyMVP AI 应用使用指南

## 概述

EasyMVP 的 AI 应用是一个 **AI 协作开发平台**，核心理念是：你描述需求，AI 团队（架构师、实现者、审核者）协作完成开发任务。

整个 AI 功能分两大块：

1. **AI 配置管理** — 配置 AI 供应商、密钥、模型（管理后台菜单：AI 管理）
2. **MVP 项目** — 创建项目，与 AI 对话，自动拆分任务并执行（管理后台菜单：MVP 项目）

---

## 一、AI 配置管理（先配好才能用）

使用 AI 功能前，需要按 **供应商 → 套餐 → 模型** 的顺序配置。

### 1.1 添加 AI 供应商

**路径**: AI 管理 → 供应商管理

供应商是 AI 服务的来源方。系统支持以下供应商：

| 供应商 | 代码 | 类型 | 说明 |
|--------|------|------|------|
| OpenAI | openai | openai_compatible | GPT 系列 |
| Anthropic | anthropic | anthropic | Claude 系列 |
| DeepSeek | deepseek | openai_compatible | DeepSeek 系列 |
| 通义千问 | qwen | openai_compatible | 阿里云 |
| 豆包 | doubao | openai_compatible | 字节跳动 |
| 文心一言 | ernie | baidu | 百度 |
| 讯飞星火 | spark | xfyun | 科大讯飞 |
| 智谱 GLM | glm | openai_compatible | 智谱 AI |
| Moonshot | moonshot | openai_compatible | 月之暗面（Kimi） |
| Yi | yi | openai_compatible | 零一万物 |
| Google | google | google | Gemini 系列 |
| Ollama | ollama | openai_compatible | 本地模型 |

**操作步骤**：
1. 点击「新增」按钮
2. 填写供应商名称（如"OpenAI"）
3. 选择供应商代码（如 `openai`）
4. 选择 Provider 类型（如 `openai_compatible`）
5. 填写 API 基础地址（如 `https://api.openai.com/v1`）
6. 状态设为「启用」
7. 保存

> **提示**：大多数国产模型都兼容 OpenAI 接口格式，类型选 `openai_compatible` 即可。

### 1.2 添加套餐（API 密钥）

**路径**: AI 管理 → 套餐管理

套餐用来管理 API 密钥。一个供应商可以有多个套餐（比如不同账号、不同额度的 Key）。

**操作步骤**：
1. 点击「新增」
2. 选择关联的供应商
3. 填写套餐名称（如"主账号"、"测试账号"）
4. 填写 API Key（系统会加密存储，列表只显示前 6 位）
5. 如果供应商需要 API Secret（如百度、讯飞），一并填写
6. 状态设为「启用」
7. 保存

### 1.3 添加模型

**路径**: AI 管理 → 模型管理

模型是实际调用的 AI 模型，需要关联到一个套餐。

**操作步骤**：
1. 点击「新增」
2. 选择关联的套餐（会自动带出供应商）
3. 填写模型显示名称（如"GPT-4o"）
4. 填写模型代码（即 API 调用时的 model 参数，如 `gpt-4o`）
5. 选择模型能力：
   - `chat` — 通用对话
   - `reasoning` — 推理能力强
   - `coding` — 编程能力强
6. 设置最大输出 Token（如 4096）
7. 设置上下文窗口大小（如 128000）
8. 是否支持流式输出（一般选「是」）
9. 可选填默认角色提示词
10. 保存

> **配置完成检查清单**：
> - [ ] 至少添加了 1 个启用状态的供应商
> - [ ] 至少添加了 1 个启用状态的套餐，且 API Key 正确
> - [ ] 至少添加了 1 个启用状态的模型

## 二、MVP 项目（核心功能）

MVP 项目是 AI 协作的核心。你创建一个项目，AI 团队会帮你从设计到实现。

### 2.1 项目生命周期

```
创建项目 → 设计阶段（与架构师对话）→ 确认方案 → 执行阶段（AI 自动执行任务）→ 完成
                ↑                                              ↓
                └──────────── 暂停（可随时暂停回到设计阶段）←─────┘
```

项目状态说明：

| 状态 | 含义 | 可以做什么 |
|------|------|-----------|
| designing | 设计中 | 与架构师对话，讨论方案 |
| running | 执行中 | AI 团队正在自动执行任务 |
| paused | 已暂停 | 暂停执行，可继续对话调整 |
| completed | 已完成 | 所有任务执行完毕 |
| failed | 失败 | 执行出错 |

### 2.2 创建项目

**路径**: MVP 项目 → 项目管理 → 新增

**关键字段**：
- **项目名称** — 简要描述这个 MVP 要做什么
- **架构师模型** — 选择一个 AI 模型作为架构师（建议选推理能力强的模型）

创建后，系统会自动：
1. 初始化一个架构师对话
2. 项目进入 `designing`（设计中）状态

### 2.3 与架构师对话（设计阶段）

**路径**: 在项目列表点击进入对话页面

这是最核心的交互环节。你在这里告诉架构师：
- 你想做什么功能
- 有什么技术要求
- 有什么限制条件

**架构师会**：
- 理解你的需求
- 拆分成具体的开发任务
- 给出实施方案

**对话界面功能**：
- 支持实时流式输出（AI 回复逐字显示）
- 显示当前项目状态
- 显示草稿任务数（架构师已拆分但未确认的任务）

**技巧**：
- 描述需求时尽量具体，包含功能细节、技术约束
- 可以多轮对话逐步细化方案
- 不满意架构师方案可以直接说，让它重新设计

### 2.4 确认方案并执行

当架构师拆分出任务后，对话页面会显示「草稿任务数」。

确认方案后：
1. 草稿任务转为待执行状态
2. 项目状态变为 `running`
3. 调度器自动分配任务给 AI 执行
4. 执行者调用 AI 模型完成每个任务
5. 审核者检查执行结果

**执行过程中你可以**：
- 在对话页面查看进展
- 暂停项目（回到设计阶段，可调整方案）
- 查看任务列表和任务日志

### 2.5 项目角色

每个项目有 4 种 AI 角色，各司其职：

| 角色 | 类型 | 职责 | 建议模型 |
|------|------|------|---------|
| 架构师 | architect | 理解需求、拆分任务、设计方案 | 推理能力强的模型 |
| 实现者 | implementer | 执行具体开发任务 | 编程能力强的模型 |
| 审核者 | auditor | 检查代码质量和正确性 | 通用对话模型即可 |
| 协调者 | coordinator | 协调各角色工作 | 通用对话模型即可 |

每个角色可以配置：
- 使用的 AI 模型
- 角色等级（lite/pro/max）
- 系统提示词（自定义角色设定）

**路径**: MVP 项目 → 项目角色

## 三、后台引擎说明

> 了解这些可以帮你更好地理解系统行为，非必读。

| 引擎 | 功能 |
|------|------|
| ChatEngine | 对话引擎，处理消息发送和 AI 回复 |
| Scheduler | 调度器，管理任务执行顺序 |
| Executor | 执行器，调用 AI 执行具体任务 |
| TaskParser | 任务解析器，从架构师输出中提取任务 |
| ContextCompressor | 上下文压缩，长对话自动摘要 |
| BugLoop | Bug 处理循环，发现问题自动修复 |
| Watchdog | 监控引擎，监控任务执行状态 |
| SSEHub | SSE 推送中心，实现实时流式输出 |

## 四、常见问题

### Q: AI 没有回复/报错？
1. 检查 AI 配置：供应商、套餐、模型是否都启用
2. 检查 API Key 是否正确、是否有余额
3. 检查 API 基础地址是否正确
4. 查看后端日志排查

### Q: 如何更换模型？
在项目角色管理中修改对应角色的模型即可，下次对话会使用新模型。

### Q: 支持本地模型吗？
支持。添加 Ollama 供应商，API 基础地址填本地 Ollama 地址（如 `http://localhost:11434/v1`），然后正常添加套餐和模型即可。

### Q: 对话太长会怎样？
系统内置了上下文压缩引擎，会自动将过长的对话历史压缩为摘要，不会因为对话过长导致 token 超限。

### Q: 可以同时运行多个项目吗？
可以。每个项目独立运行，互不影响。

## 五、快速上手流程

```
1. AI 管理 → 添加供应商（如 OpenAI）
2. AI 管理 → 添加套餐（填入 API Key）
3. AI 管理 → 添加模型（如 gpt-4o）
4. MVP 项目 → 创建项目，选择架构师模型
5. 进入对话页面，告诉架构师你想做什么
6. 架构师拆分出任务后，确认方案
7. AI 团队自动执行，等待完成
```
