# 多任务依赖回归样例

场景编码：`multi_task_dependency`

目标：

- 覆盖批次门控、依赖推进和回写汇总
- 验证项目轨迹页与任务回放对多任务链路的可观测性

建议仓库形态：

- 一个最小 Go 或 Node 服务
- 至少包含 `README.md`、`main` 入口、一个配置文件

建议任务拆分：

1. 批次 1：补充配置与基础文档
2. 批次 2：实现核心逻辑
3. 批次 3：补充测试或验收说明

关键观察点：

1. 前一批次未完成时，后一批次不能启动
2. 项目轨迹页可展示阶段、事件、交付模式和回写状态聚合
3. 执行控制台可按批次展示任务
4. 任务回放可看到日志、交接、证据与决策
